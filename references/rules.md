# Skill 开发规范规则

## 1. 目录结构规范

### 必选文件
- `SKILL.md` - 每个skill的入口文件，必须存在

### 可选目录
| 目录 | 用途 | 说明 |
|------|------|------|
| `scripts/` | 可执行脚本 | Python、Bash等脚本 |
| `references/` | 参考文档 | 按需加载的详细文档 |
| `assets/` | 资源文件 | 模板、图片、字体等 |

### 禁止文件
- `README.md`
- `CHANGELOG.md`
- `INSTALLATION_GUIDE.md`
- `QUICK_REFERENCE.md`
- `TODO.md`
- `NOTES.md`

**原因**: Skill不应包含辅助文档，所有必要信息应在SKILL.md中。

## 2. Frontmatter规范

### 必选字段
```yaml
---
name: skill-name
description: 技能描述（包含功能和使用场景）
---
```

### 可选字段
- `license` - 许可证类型
- `allowed-tools` - 允许使用的工具列表
- `metadata` - 元数据对象

### 禁止字段
除上述字段外，禁止使用其他YAML字段。

## 3. 命名规范

### name字段规则
- **字符**: 小写字母(a-z)、数字(0-9)、连字符(-)
- **格式**: hyphen-case (如 `my-skill-name`)
- **长度**: 最大64字符
- **禁止**:
  - 开头或结尾使用连字符
  - 连续连字符 (`--`)
  - 大写字母
  - 下划线或其他特殊字符

### 示例
| 合法 | 非法 |
|------|------|
| `my-skill` | `My-Skill` |
| `skill-v2` | `skill_v2` |
| `pdf-editor` | `pdfEditor` |

## 4. Description规范

### 内容要求
- **长度**: 最大1024字符
- **必须包含**:
  1. 功能描述 - 技能做什么
  2. 触发场景 - 何时使用

### 禁止内容
- HTML标签 `<` `>`
- 特殊编码

### 良好示例
```yaml
description: >
  Skill规范检查工具。扫描并验证skill是否符合开发规范，
  输出诊断报告及修复建议。触发场景：检查skill、验证规范、
  扫描skill目录、诊断报告、skill质量评估。
```

## 5. 内容质量规范

### 长度控制
- **SKILL.md正文**: 建议不超过500行
- **参考文件**: 大文件(>200行)应拆分

### 渐进式披露
将详细信息移到 `references/` 目录，SKILL.md只保留核心流程。

### 代码示例
- 使用真实代码而非伪代码
- 包含必要的边界情况处理

## 6. 资源文件规范

### Scripts
- 添加shebang行 (`#!/usr/bin/env python3`)
- 确保可执行权限
- 提供清晰的注释

### References
- 文件名使用kebab-case
- 大文件添加目录索引
- 避免深层嵌套

### Assets
- 只包含必要文件
- 使用通用格式
- 注明来源和许可证

## 7. 检查级别

| 级别 | 说明 | 扣分 |
|------|------|------|
| ERROR | 严重错误，必须修复 | -20 |
| WARN | 警告，建议修复 | -5 |
| INFO | 提示信息，可选优化 | -1 |

## 8. 评分标准

| 分数 | 等级 | 说明 |
|------|------|------|
| 90-100 | A | 优秀，符合所有规范 |
| 70-89 | B | 良好，有少量警告 |
| 50-69 | C | 一般，存在错误需修复 |
| 0-49 | F | 不合格，需要重大修复 |

## 9. 常见问题

### Q: 为什么不能有README.md?
A: Skill是自包含的，SKILL.md应包含所有必要信息。额外文档增加维护负担且容易被忽视。

### Q: 何时使用references/?
A: 当SKILL.md超过200行时，应拆分详细信息到references/。

### Q: 描述可以多长?
A: 最大1024字符。简洁描述更有价值。

### Q: 如何选择name?
A: 使用功能相关的简短名称，用连字符分隔。如 `pdf-editor`、`git-helper`。
